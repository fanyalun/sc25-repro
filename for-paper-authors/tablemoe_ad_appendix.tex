\section{Overview of Contributions and Artifacts}

\subsection{Paper's Main Contributions}

\begin{description}
\item[$C_1$] We introduce TableMoE, a training-free approximate expert lookup
and reuse framework for MoE MLLMs. Specifically, TableMoE reduces redundant
expert execution and expert-weight loading by combining approximate lookup,
online reuse, and expert-level scheduling in a CPU--GPU offloading system.
\item[$C_2$] We propose the approximate reuse mechanisms in Sections IV-B and
IV-C. Specifically, the prototype-based ALUT replaces exact matching over
continuous multimodal hidden states with compact semantic prototypes, while
the sliding-window reuse cache exploits request-local temporal and semantic
locality for more accurate in-context reuse.
\item[$C_3$] We propose the expert-level importance-aware scheduler in
Section IV-D. Specifically, it estimates expert importance at the request and
expert granularity, and uses the prefill recompute ratio to balance exact
expert recomputation with approximate reuse.
\end{description}

\section{Artifact Identification}

\newartifact

\artrel

This artifact contains the source code and released assets of TableMoE and
covers all contributions of this paper, including the TableMoE framework
($C_1$), the approximate reuse mechanisms ($C_2$), and the expert-level
scheduler ($C_3$).

\artexp

TableMoE aims to accelerate MoE MLLM inference under CPU--GPU heterogeneous
architectures while preserving generation quality. First, we expect TableMoE
to achieve lower TTFT and TPOT than the representative baselines in the default
configuration. Second, we expect the prototype-based ALUT and sliding-window
reuse cache to provide higher accuracy than the lossy baseline that directly
skips the same expert computation, and the final TableMoE system should keep
accuracy degradation within 1\% of the Transformers baseline. Third, we expect
the expert-level importance-aware scheduler to further reduce TTFT and TPOT,
while improving the Decoding Expert Cache Hit Rate over the compared variants
and baselines.

\arttime

\noindent\textbf{Setup:} 10--30 minutes. This includes cloning the repository,
creating the Python environment, and installing the required dependencies.

\noindent\textbf{Execution:} about 1--2 hours for the performance and
cache-hit stages and about 2--2.5 hours for the accuracy stage in the quick
reproduction workflow. Preparing the released offline tables usually takes
10--20 minutes; rebuilding them locally instead of downloading the released
version adds about 2--3 hours to the execution stage.

\noindent\textbf{Analysis:} less than 5 minutes. The scripts write the main
comparison tables, manifest, and summary files directly to the result
directories during execution.

\artin

\artinpart{Hardware}

All experiments are conducted on a CPU--GPU
heterogeneous platform using one NUMA node of an Intel Xeon Gold 5318Y CPU
(24 cores), 256~GB DDR4 memory, and one NVIDIA A100 PCIe GPU with 80~GB HBM,
connected via PCIe~4.0~$\times$16.

\artinpart{Installation and Deployment}

We use a Conda-based Python environment as the base execution environment.
Please follow the instructions in Sections 2--4 of the repository README to
set up the environment, install the required Python dependencies, and prepare
the model-specific packages for Qwen3-VL-30B-A3B-Instruct or DeepSeek-VL2. If
a prebuilt flash-attn wheel is unavailable for the local environment, the same
CUDA 12.8 toolchain must be used to build it locally.

\artcomp

The artifact workflow is:
\begin{description}
\item[$T_1$] Prepare the environment and install the dependencies following
Section 2 of the repository README.
\item[$T_2$] Prepare the required inputs following Section 3.1 of the
repository README, including the models, datasets, and offline tables.
\item[$T_3$] Run the evaluation workflow under different expert cache-ratio
and prefill recompute ratio settings to compare TableMoE with the
representative baselines. This step measures the main performance, accuracy,
and decode-only cache-hit-rate metrics reported in the paper.
\end{description}

The workflow dependency is as follows:
$T_1 \rightarrow T_2 \rightarrow T_3$.

\artout

The outputs include a human-readable quick-reproduction report, a
machine-readable manifest, summary tables for performance and accuracy, and
the corresponding intermediate evaluation results. These outputs cover the base
accuracy comparison, the performance comparison across different cache ratios,
and the TableMoE comparison across different prefill recompute ratio settings
in terms of performance, accuracy, and decode-only cache hit rate.
