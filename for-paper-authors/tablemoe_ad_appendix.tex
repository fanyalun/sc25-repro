\section{Overview of Contributions and Artifacts}

\subsection{Paper's Main Contributions}

\begin{description}
\item[$C_1$] We propose TableMoE, a training-free multimodal lookup framework
that replaces redundant MoE expert computation with table lookup and expert
reuse in MoE MLLMs.
\item[$C_2$] We evaluate TableMoE against representative baselines under
different expert cache ratios and show that TableMoE delivers consistently
better latency across the cache-ratio range.
\item[$C_3$] We analyze the effect of the prefill recomputation ratio on
TableMoE and show the tradeoff between latency, cache-hit behavior, and
accuracy under different prefill recomputation settings.
\end{description}

\section{Artifact Identification}

\newartifact

\artrel

This artifact contains the source code and released assets of TableMoE and
covers all contributions of this paper, including $C_1$, $C_2$, and $C_3$.

\artexp

This artifact is expected to reproduce the main latency and accuracy trends
reported in the paper. First, for $C_1$, the artifact should successfully run
the training-free TableMoE workflow on the supported models and datasets
provided by the repository. Second, for $C_2$, the artifact should reproduce
the performance comparison between TableMoE and the representative baselines
under different expert cache ratios. Third, for $C_3$, the artifact should
reproduce the TableMoE comparison under different prefill recomputation
keep-rate settings in terms of TTFT, TPOT, accuracy, and decode-only cache hit
rate.

The repository writes intermediate quick-reproduction outputs under
\path{perf_results/quick_reproduction/} and
\path{acc_results/quick_reproduction/}. The final quick-reproduction artifacts
are
\path{perf_results/quick_reproduction/summary/quick_reproduction_report.md}
and
\path{perf_results/quick_reproduction/summary/quick_reproduction_manifest.json}.
For arbitrary-parameter runs, \texttt{eval\_perf} writes
\path{perf_results/<result_dir_name>/summary/\{perf_table,simple_accuracy_table\}.\{json,csv,md\}},
and \texttt{eval\_acc} writes
\path{acc_results/<result_dir_name>/summary/accuracy_table.\{json,csv,md\}}.

\arttime

Setup: 10--30 minutes. This includes cloning the repository, creating the
Python environment, and installing the required dependencies.

Execution: about 1--1.5 hours for the performance and cache-hit stages, about
2--2.5 hours for the accuracy stage, and only a few additional minutes for the
final report generation in the quick reproduction workflow. Preparing the
released offline tables usually takes 10--20 minutes; rebuilding them locally
instead of downloading the released version adds about 2--3 hours to the
execution stage.

Analysis: less than 5 minutes. The scripts write the main comparison tables,
manifest, and summary files directly to the result directories during
execution.

\artin

\artinpart{Hardware}

All experiments are conducted on a CPU--GPU
heterogeneous platform using one NUMA node of an Intel Xeon Gold 5318Y CPU
(24 cores), 256~GB DDR4 memory, and one NVIDIA A100 PCIe GPU with 80~GB HBM,
connected via PCIe~4.0~$\times$16.

\artinpart{Software}

The software environment used for evaluation is configured as follows:
Ubuntu@22.04.4, Python@3.10.18, CUDA@12.8, NVIDIA Driver@580.95.05,
PyTorch@2.8.0+cu128, Transformers@4.57.0, FlashAttention@2.7.3, and
Triton@3.4.0. When running DeepSeek-VL2, switch Transformers to 4.38.0 and
install xformers@0.0.32.post1 as described in the repository README.

\artinpart{Datasets / Inputs}

We use two MoE MLLMs and five public multimodal benchmarks in our evaluation,
as listed below. The models and datasets can be downloaded and prepared using
the scripts provided in the repository README.
\begin{itemize}
\item Models:
  \begin{itemize}
  \item Qwen3-VL-30B-A3B-Instruct
  \item DeepSeek-VL2
  \end{itemize}
\item Datasets:
  \begin{itemize}
  \item RealWorldQA
  \item MMBench\_DEV\_EN\_V11
  \item AI2D\_TEST
  \item ScienceQA\_TEST
  \item POPE
  \end{itemize}
\end{itemize}

\artinpart{Installation and Deployment}

We use a Conda-based Python environment as the base execution environment.
Please follow the instructions in Sections 2--4 of the repository README to
set up the environment, install the required Python dependencies, and prepare
the model-specific packages for Qwen3-VL-30B-A3B-Instruct or DeepSeek-VL2. If
a prebuilt flash-attn wheel is unavailable for the local environment, the same
CUDA 12.8 toolchain must be used to build it locally.

\artcomp

The artifact workflow is:
\begin{description}
\item[$T_1$] Prepare the environment and install the dependencies following
Section 2 of the repository README.
\item[$T_2$] Prepare the required inputs following Section 3.1 of the
repository README, including the models, datasets, and offline tables.
\item[$T_3$] Run \texttt{run\_quick\_reproduction.sh}. This workflow executes
the \texttt{perf}, \texttt{cache\_hit}, \texttt{acc}, and \texttt{report}
stages and writes the quick-reproduction outputs under
\path{perf_results/quick_reproduction/} and
\path{acc_results/quick_reproduction/}.
\item[$T_4$] For arbitrary-parameter sweeps, run \texttt{eval\_perf} and
\texttt{eval\_acc} following Section 4 of the repository README. These scripts
write the full-reproduction summary tables under the corresponding
\path{perf_results/<result_dir_name>/summary/} and
\path{acc_results/<result_dir_name>/summary/} directories.
\end{description}

The workflow dependency is as follows:
$T_1 \rightarrow T_2 \rightarrow T_3$, and $T_2 \rightarrow T_4$.
If \texttt{run\_quick\_reproduction.sh} is invoked with a subset of
\texttt{STEPS}, only the corresponding subset of quick-reproduction artifacts
is generated. The complete \path{quick_reproduction_report.md} is produced only
when the \texttt{report} stage executes successfully.

\artout

The main outputs are written under \path{perf_results/<result_dir_name>/} and
\path{acc_results/<result_dir_name>/}. For the quick reproduction workflow,
the intermediate outputs are stored under
\path{perf_results/quick_reproduction/} and
\path{acc_results/quick_reproduction/}, and the final top-level artifacts are
\path{perf_results/quick_reproduction/summary/quick_reproduction_report.md}
and
\path{perf_results/quick_reproduction/summary/quick_reproduction_manifest.json}.
The markdown report is the primary human-readable artifact, and the manifest is
the machine-readable index of the quick-reproduction experiment roots and the
configuration used to render the final report. The final markdown report
contains the base accuracy comparison, the cache-ratio performance comparison,
and the keep-rate comparison of TableMoE in terms of performance, accuracy,
and decode-only cache hit rate.

For arbitrary-parameter runs, the repository writes the following summary
tables:
\begin{itemize}
\item \path{perf_results/<result_dir_name>/summary/\{perf_table,simple_accuracy_table\}.\{json,csv,md\}}
\item \path{acc_results/<result_dir_name>/summary/accuracy_table.\{json,csv,md\}}
\end{itemize}

These outputs are used to compare the baseline methods and TableMoE across the
supported models, datasets, cache ratios, and prefill recomputation
keep-rate settings.
