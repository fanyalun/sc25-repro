\section{Overview of Contributions and Artifacts}

\subsection{Paper's Main Contributions}

\begin{description}
\item[$C_1$] We introduce a prototype-based approximate lookup table (ALUT)
mechanism for accelerating MoE-based MLLMs by replacing redundant expert
execution with approximate lookup and reuse.
\item[$C_2$] We design a sliding-window reuse cache that exploits in-context
locality during decoding to improve approximation fidelity and preserve model
accuracy.
\item[$C_3$] We propose an expert-level importance-aware scheduler that
selects which experts should be recomputed exactly and which can be served via
approximate reuse.
\item[$C_4$] We present an experimental evaluation showing that TableMoE
reduces TTFT and TPOT while maintaining high task accuracy without retraining.
\end{description}

\section{Artifact Identification}

\newartifact

\artrel

This artifact contains the implementation of TableMoE, including the offline
ALUT construction pipeline, the runtime reuse/cache/scheduling logic, and the
evaluation scripts used to measure latency and accuracy. The same artifact is
used to support all four contributions of the paper.

\artexp

This artifact is expected to reproduce the main latency and accuracy trends
reported in the paper. The quick reproduction path generates a TTFT/TPOT
summary from \texttt{eval\_perf}, a lightweight accuracy comparison from
\texttt{eval\_perf}, and an accuracy summary from \texttt{eval\_acc}. These
outputs correspond to the paper's main evaluation results.

\arttime

\begin{description}
\item[Setup] 10--30 minutes. This covers cloning the repository, creating the
Python environment, and installing the dependencies.
\item[Optional offline-table preparation] Downloading the released offline
tables takes 10--20 minutes. Rebuilding the offline tables locally takes about
2--3 hours, depending on storage speed and the time required to profile and
cluster routed hidden states.
\item[Execution] Running \texttt{eval\_perf} takes about 20--30 minutes.
Running \texttt{eval\_acc} takes about 2 hours.
\item[Analysis] Less than 5 minutes. The scripts write the main comparison
tables directly to the result directories.
\end{description}

\artin

\artinpart{Hardware}

All experiments are conducted on a CPU--GPU
heterogeneous platform using one NUMA node of an Intel Xeon Gold 5318Y CPU
(24 cores), 256~GB DDR4 memory, and one NVIDIA A100 PCIe GPU with 80~GB HBM,
connected via PCIe~4.0~$\times$16.

\artinpart{Software}

The validated software stack is:
\begin{itemize}
\item Ubuntu 22.04.4 LTS (\url{https://ubuntu.com/})
\item Python 3.10.18 (\url{https://www.python.org/})
\item CUDA 12.8 (\url{https://developer.nvidia.com/cuda-toolkit})
\item NVIDIA Driver 580.95.05 (\url{https://www.nvidia.com/drivers})
\item PyTorch 2.8.0+cu128 (\url{https://pytorch.org/})
\item torchvision 0.23.0+cu128
(\url{https://pytorch.org/vision/stable/index.html})
\item transformers 4.57.0
(\url{https://github.com/huggingface/transformers})
\item flash-attn 2.7.3
(\url{https://github.com/Dao-AILab/flash-attention})
\item triton 3.4.0 (\url{https://github.com/triton-lang/triton})
\item TableMoE repository
(\url{https://github.com/fanyalun/tablemoe-sc26.git})
\end{itemize}
The quick reproduction path does not require CUTLASS, MoE-Infinity,
Pregated-MoE, or a local judge model.

\artinpart{Datasets / Inputs}

The artifact requires the model checkpoint, the benchmark TSV files, and the
optional released offline tables described in Sections 1--3 of the repository
README.
\begin{itemize}
\item Model checkpoint
\item Benchmark TSV files
\item Optional released offline tables
\end{itemize}

The remaining installation and input details follow Sections 1--3 of the
repository README.

\artinpart{Installation and Deployment}

Please follow Section 2 of the repository README to create the Python
environment and install the required dependencies. If a prebuilt flash-attn
wheel is unavailable for the local environment, the same CUDA 12.8 toolchain
must be used to build it locally.

\artcomp

The quick reproduction workflow is:
\begin{description}
\item[$T_1$] Prepare the environment and install the dependencies following
Section 2 of the repository README.
\item[$T_2$] Prepare the required inputs following Section 3.1 of the
repository README, including the model checkpoint, benchmark TSV files, and
optionally the released offline tables.
\item[$T_3$] Run the quick reproduction workflow in Section 3.2 of the
repository README. This executes \texttt{eval\_perf} and \texttt{eval\_acc}
and writes the corresponding summary tables.
\end{description}

The workflow dependency is $T_1 \rightarrow T_2 \rightarrow T_3$.

\artout

The main outputs are written directly to disk under the default summary
directories:
\begin{itemize}
\item \path{perf_results/default/summary/perf_table.\{json,csv,md\}}
\item \path{perf_results/default/summary/simple_accuracy_table.\{json,csv,md\}}
\item \path{acc_results/default/summary/accuracy_table.\{json,csv,md\}}
\end{itemize}

The latency comparison is taken from \texttt{eval\_perf}, while the accuracy
comparison in this appendix is taken from \texttt{eval\_acc}. These result
files are the primary outputs used to compare TableMoE with the baselines in
the quick reproduction path.
